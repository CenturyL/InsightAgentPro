# 企业级本地知识智能助手 (Local Knowledge Agent API)

本项目旨在结合大语言模型前沿技术（LangChain、RAG、Agent），使用纯本地化的大模型方案（保护数据隐私），并搭配目前热门的异步高性能 Web 框架（FastAPI），打造一个能够写在简历上的亮点实战项目。

## 1. 项目需求与背景

在很多垂直以及对数据隐私敏感的应用场景（如政务、金融、医疗或企业内网），不允许将机密数据（PDF、Word档等）上传至外网环境。因此：
* 必须支持 **本地 LLM 接入**（例如使用 Ollama 跑 Qwen/Llama 模型）；
* 需要建立一个 **本地知识库** (RAG)，辅助解决大模型训练数据过期和幻觉缺陷；
* 业务不单是问答，往往还需要执行特定的任务操作，因此引入 **Agent 与 Tool 机制**（如查询本地私有库、调用天气 API、执行网页搜索）；
* 为了应对大模型生成速度慢的问题，前端需要看到打字机效果，所以 API 必须支持 **Server-Sent Events (SSE) 流式输出**。

## 2. 技术栈选型

* **大模型驱动引擎**：Ollama (运行本地开源模型，如 Qwen2.5/Llama3 等)
* **LLM 编排框架**：LangChain (与 LangChain 表达式语言 LCEL)
* **Embedding (向量化模型)**：BGE-M3 (或 Ollama 提供的 nomic-embed-text)
* **向量数据库**：ChromaDB (本地轻量级持久化向量库)
* **后端 API 框架**：FastAPI + Uvicorn (支持原生的 async 异步和 SSE，高并发量)
* **数据校验模型**：Pydantic V2

## 3. 核心 API 接口设计规划

系统包含三大核心接口，分别对应：知识注入、RAG对话、Agent协同。

### 3.1 知识上传接口
* **路由**: `POST /knowledge/upload`
* **功能**: 用户上传 PDF、TXT、Markdown 文件。系统通过 LangChain DocumentLoaders 加载文本，利用 RecursiveCharacterTextSplitter 进行切分，最终利用本地 Embedding 模型生成向量数据并写入 ChromaDB。
* **返回值**: 返回处理成功的块数量及文档 ID。

### 3.2 知识库问答接口 (RAG Stream)
* **路由**: `POST /chat/rag`
* **功能**: 接收用户的自然语言提问，实时使用 ChromaDB 从本地知识库中召回前 K 个最相关的文本块，并配合设定好的 Prompt，以流式形式（StreamingResponse）回复给用户。结合 Memory组件实现带记忆的对话。

### 3.3 智能体 Agent 接口
* **路由**: `POST /chat/agent`
* **功能**: 接收复杂的用户任务指令，LLM 自行决定是否动用工具。
    * Tool 1: `RagSearchTool` (到我们自己的文档里查资料)
    * Tool 2: `WebSearchTool` (借助 DuckDuckGo / Tavily 的 API 进行外网最新内容检索)
    * Tool 3: `CalculatorTool` (时间或数学运算代理)

## 4. 项目总体架构与模块划分

```text
local_agent_api/
├── main.py              # FastAPI 启动入口与主路由注册
├── requirements.txt     # 项目依赖 (fastapi, langchain, chromadb 等)
├── data/                # 存放本地 Chroma 数据库文件和上传的临时文件
├── core/                # 核心基础组件集成
│   ├── __init__.py
│   ├── config.py        # 全局配置参数（系统提示词、模型名称、数据库路径等）
│   ├── llm.py           # 连接 Ollama 本地模型的工厂函数
│   └── embedding.py     # 获取本地 Embedding 模型的单例
├── services/            # 核心业务逻辑实现 (LangChain LCEL及链的构建)
│   ├── __init__.py
│   ├── rag_service.py   # RAG 相关的所有链 (文档切分存储、检索、生成)
│   └── agent_service.py # Agent 及 Tools 的编排逻辑
└── api/                 # FastAPI 的 HTTP 交互层
    ├── __init__.py
    ├── routes.py        # 对应的 /chat 和 /knowledge 接口路由处理
    └── schemas.py       # 请求和响应的 JSON 数据结构定义 (Pydantic models)
```

## 5. 项目实现排期建议 (Roadmap)

* **Step 1：安装依赖与 Ollama 环境就绪**（能在终端与本地模型打通）。
* **Step 2：构建 Core 模块**（将 `llm.py` 和 `embedding.py` 打通，验证生成和向量化功能）。
* **Step 3：攻克 RAG 骨架**（在 `rag_service.py` 实现文件入库和问答逻辑，测试切块和召回）。
* **Step 4：FastAPI 与流式输出接入**（连接 `main.py` 和 `api/routes.py`，实现 Postman 流式流出）。
* **Step 5：Agent 的集成**（最后进阶，为大模型配备搜索和查阅私有库的工具）。
