# LangChain 进阶学习路线指南

从掌握 LangChain 基础（Prompt、基础 LLMChain、基础检索）到具备工程化实战能力，你需要重点攻克以下五个关键领域。这份指南将为你提供清晰的学习路径和核心知识点。

## 1. 高级 RAG 技术 (Advanced RAG)

基础 RAG（文本切块 -> 向量化 -> 相似度检索）在面对复杂业务时往往会出现召回精度低、上下文缺失等问题。高级 RAG 旨在解决这些痛点。

### 核心知识点：
* **Multi-Query Retriever (多查询检索)**
  * **原理**：利用 LLM 将用户的原始问题改写成多个不同表达方式的相似问题，然后分别进行检索，最后对结果取并集。
  * **解决的问题**：用户提问表述不清或用词与文档不一致导致的召回失败。
* **Parent Document Retriever (父文档检索器)**
  * **原理**：在建库时，将大文档（父）切分为小文档块（子）。向量数据库对小文档块做 Embedding 和检索，但当命中某个小块时，返回给 LLM 生成的是它对应的整个大块（或原文档）。
  * **解决的问题**：“存小段搜大段”——小段确保检索的高精度，大段确保喂给 LLM 的上下文完整性。
* **Self-Querying Retriever (自查询检索)**
  * **原理**：利用 LLM 将用户的自然语言（例如：“2023年关于AI的科幻电影有哪些？”）解析为包含搜索词（“科幻电影”）和元数据过滤条件（`{"year": 2023, "topic": "AI"}`）的结构化查询。
  * **解决的问题**：不仅能做语义搜索，还能基于文档 Metadata 进行精准过滤。
* **Contextual Compression (上下文压缩)**
  * **原理**：召回大量文档后，再使用一个 LLM 提取每个文档中真正与问题相关的部分进行压缩，甚至剔除无关文档（Rerank机制）。

## 2. Agent 与动态 Tool 机制

让 LLM 从“文本生成器”变成能干活的“智能体”。这不仅是简历上最吸睛的点，也是未来大模型应用的主流。

### 核心知识点：
* **核心范式：ReAct (Reason + Act)**
  * 学习 Agent 的运行机制：思考（我要做什么） -> 行动（调用工具） -> 观察（获取工具结果） -> 思考（拿到结果后下一步怎么做）... 直到完成任务。
* **工具的定义与绑定 (Tools)**
  * 掌握使用 `@tool` 装饰器或继承 `BaseTool` 类来编写自定义工具。
  * **实战案例**：编写一个能查询数据库的 Tool，一个能请求外部天气的 Tool。
* **OpenAI Tools Agent / Function Calling**
  * 学习大模型原生支持的函数调用能力（Function Calling），这是目前最稳定、最高效的 Agent 实现方式。
* **LangGraph (LangChain v0.1+ 推荐)**
  * 逐渐了解用图（Graph）结构来替代传统的 AgentExecutor，实现更复杂、可控的多 Agent 循环和状态机流转。

## 3. 记忆与状态持久化 (Memory & Persistence)

如何让你的 AI 助手不仅能在当前回话中记住上下文，还能跨时间、跨设备记住用户偏好。

### 核心知识点：
* **BaseChatMessageHistory 接口**
  * 理解 LangChain 底层的消息历史抽象类。
* **数据库持久化集成**
  * 学习将记忆存储到外置组件中：
    * `RedisChatMessageHistory` (适合需要设置过期的会话缓冲)
    * `PostgresChatMessageHistory` / `SQLite` (适合长久保存的用户档案)
* **高级记忆类型**
  * 除了基础的 Buffer Memory，了解 `ConversationSummaryMemory`（当对话过长时，LLM 先对之前的对话进行总结压缩以节省 Token）。

## 4. 流式输出与异步处理 (Streaming & Async)

在本地部署/工程化中，LLM 的生成速度普遍较慢，阻塞式等待会极大影响用户体验。掌握流式输出是必须跨越的门槛。

### 核心知识点：
* **LCEL 的异步与流式接口**
  * 掌握新版 LangChain 表达式语言 (LCEL) 的 `astream()`（异步流式生成）和 `astream_events()`（不仅流式吐字，还能监听 Agent 调用工具的过程）。
* **搭配 FastAPI 实现 SSE**
  * 学习如何将 LangChain 的 async 生成器，通过 FastAPI 的 `StreamingResponse` 包装成 Server-Sent Events (SSE) 协议接口。
  * 这能让你开发出像 ChatGPT 那样“打字机”效果的后端。

## 5. 部署与监控 (LangServe & LangSmith)

证明你不仅会在本地跑脚本，还懂得如何将大模型应用推向生产环境。

### 核心知识点：
* **LangServe**
  * LangChain 官方开源的轻量级部署工具（基于 FastAPI）。
  * 学习通过一两行代码把定义好的 LCEL Chain 或 Agent 直接转换为符合规范的 RESTful API（涵盖 invoke, stream, batch 等接口）。
* **LangSmith (大杀器)**
  * 学习接入 LangSmith 平台进行 Trace（调用链追踪）。
  * **重要性**：当 AI 给错答案、报错或在某一步卡住时，你可以通过 LangSmith 的可视化面板看到它是怎么思考的、传了什么 Prompt 给底层模型、消耗了多少 Token 以及调用链的时延。这是简历上极具亮点的工程级调试能力。
